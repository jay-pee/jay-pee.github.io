Here is a hard truth: As a software develop you will have most likely write code this day, that no one will every use again 1 year from now. You basically do stuff now that is a wast of time one year from now. Your code today is the lagacy code for someone else which they will complain about:

- "Hier hat sich auch jemand in einer ganz besonderen Art verewigt"
- "Why is everything so complicated?"
- "Ok I am done with it. Lets do a rewrite"

What are the priorities

- It need to be fast
- The usecase must be christal clear
	- The feature set must be limited and easy to grasp
- No unecessary clicks needed
- Setup must be easy
- It must be robust, fail with grace
	- Failer should explain why
- UI should be put some thoughts in
	- People don't read 90% of the text they are presented. Guidance to the parts that are important is useful
